# Best Practices

## 1. 避免破坏性字段替换

protobuf 不支持就地改字段类型并保持兼容。迁移应采用“新增字段 + 过渡期并存 + 最后清理”。

## 2. 统一时间基准

内部转换统一使用 UTC 毫秒作为中间语义，避免：

- 秒/毫秒混淆
- 时区偏差
- 前端格式化不一致

## 3. 过渡期必须双向验证

每个时间字段都要验证：

- 新字段可读
- 旧字段回退可读
- 两者代表同一时刻

## 4. 持久化层保持最小变更

在协议迁移批次不要顺带重构存储格式。先把协议消费打通，再评估是否需要存储层升级。

## 5. 前端适配集中化

不要在多个组件散落解析逻辑。通过单一适配器函数处理 `Timestamp | int64`，减少回归面。

## 6. 先覆盖进度域，再扩展到文档时间字段

优先处理 `SyncProgressState` 相关字段。`IndexDocument.modified_at/created_at` 体量更大，建议下一阶段单独评估。

## 7. 通过观察窗口决定旧字段下线

旧字段清理必须基于实际客户端升级覆盖率，不应在引入新字段的同一批次进行。
